## О проекте

Task-Chat — это веб-приложение для управления задачами на медицинские исследования (CT, MR, PET) и организации коммуникации между врачами и физиками.

---

## Роли пользователей

В системе существует **три роли**:

| Роль | Описание | Права доступа |
|------|----------|---------------|
| **admin** | Администратор системы | Полный доступ ко всем функциям системы |
| **doctor** | Врач | Создание задач, просмотр своих задач, чат с физиками |
| **physicist** | Физик | Просмотр всех задач, изменение статуса, чат с врачами |

### Детальные права доступа

#### Врач (doctor)

**Может:**
- ✅ Создавать новые задачи на исследования
- ✅ Просматривать только свои задачи (которые создал)
- ✅ Писать сообщения в чат своих задач
- ✅ Прикреплять файлы к своим задачам
- ✅ Просматривать пользователей онлайн
- ✅ Использовать личные чаты с другими пользователями

**Не может:**
- ❌ Просматривать задачи других врачей
- ❌ Изменять статус задач (только физики и админ)
- ❌ Редактировать задачи после создания
- ❌ Удалять задачи
- ❌ Регистрировать пользователей

#### Физик (physicist)

**Может:**
- ✅ Просматривать все задачи системы
- ✅ Изменять статус задач (pending, in_progress, completed, cancelled)
- ✅ Писать сообщения в чат любой задачи
- ✅ Прикреплять файлы к любым задачам
- ✅ Просматривать пользователей онлайн
- ✅ Использовать личные чаты

**Не может:**
- ❌ Создавать новые задачи
- ❌ Редактировать детали задачи (только статус)
- ❌ Удалять задачи
- ❌ Регистрировать пользователей
- ❌ Доступ к админ-панели

#### Администратор (admin)

**Может:**
- ✅ Всё, что могут врач и физик
- ✅ Доступ к админ-панели
- ✅ Регистрация новых пользователей
- ✅ Редактирование любых задач
- ✅ Удаление любых задач
- ✅ Массовые операции с задачами
- ✅ Смена паролей пользователей
- ✅ Удаление пользователей
- ✅ Просмотр полной статистики системы

## Структура данных

### Пользователь (User)

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | Уникальный ID |
| username | string | Имя пользователя (уникальное) |
| password_hash | string | Хэш пароля |
| role | string | Роль: `admin`, `doctor`, `physicist` |
| first_name | string | Фамилия |
| last_name | string | Имя |
| middle_name | string | Отчество (опционально) |
| department | string | Отделение |
| created_at | datetime | Дата регистрации |
| last_seen | datetime | Последнее появление в сети |

**Отделения по умолчанию:**
- Для врачей: `РО1`, `РО2`, `РО3`, `РО4`, `РО5`
- Для физиков: `ФРО`
- Для админов: `ТП`

### Задача (Task)

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | Уникальный ID |
| title | string | Заголовок |
| description | text | Описание |
| status | string | Статус: `pending`, `in_progress`, `completed`, `cancelled` |
| priority | string | Приоритет: `low`, `medium`, `high`, `urgent` |
| research_type | string | Тип: `import_ct`, `import_ct_diag`, `mr`, `pet` |
| expert_group | string | Экспертная группа (для CT) |
| patient_card | string | Номер карты пациента |
| aria_availability | string | Доступность Aria: `new` (Новый) или `existing` (Уже есть) |
| ct_diagnostic | string | CT-диагностика (аппарат + корпус) |
| treatment | string | Лечение (аппарат + корпус) |
| breath_control | string | Контроль дыхания: `no`, `breath_hold`, `free` |
| mr_diagnostic | string | MR-диагностика |
| pet_diagnostic | string | PET-диагностика |
| doctor_id | int | ID врача-создателя |
| physicist_id | int | ID физика (по умолчанию = 1) |
| created_at | datetime | Дата создания |
| updated_at | datetime | Дата последнего изменения |
| completed_at | datetime | Дата завершения |

### Статусы задач

| Статус | Описание |
|--------|----------|
| `pending` | Ожидает — задача создана, но ещё не обрабатывается |
| `in_progress` | В работе — физик обрабатывает задачу |
| `completed` | Выполнена — задача завершена |
| `cancelled` | Отменена — задача отменена |

### Сообщения (ChatMessage)

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | Уникальный ID |
| content | text | Текст сообщения |
| task_id | int | ID задачи |
| sender_id | int | ID отправителя |
| receiver_id | int | ID получателя |
| is_read | bool | Прочитано ли |
| created_at | datetime | Дата отправки |

### Личные чаты

**PersonalChat** — чат между двумя пользователями:
- user1_id, user2_id — участники
- created_at, updated_at — даты

**PersonalMessage** — сообщение в личном чате:
- chat_id — ID чата
- sender_id — отправитель
- content — текст
- is_read — прочитано
- files — прикреплённые файлы

## Логика доступа к задачам

### Кто видит задачу

- **Врач**: видит только задачи, где `doctor_id = current_user.id`
- **Физик**: видит все задачи
- **Админ**: видит все задачи

### Кто может писать в чат задачи

- **Врач-создатель**: всегда имеет доступ
- **Физик**: имеет доступ ко всем задачам
- **Админ**: имеет доступ ко всем задачам

**Получатель сообщения:**
- Если отправляет врач → получают все физики (receiver_id = первый физик)
- Если отправляет физик → получает врач-создатель
- Если отправляет админ → получает врач-создатель

### Участники чата задачи

Участник чата = врач-создатель + все, кто отправлял сообщения в эту задачу

## API Endpoints

### Публичный endpoint

#### GET `/api/stats`

Статистика системы (не требует аутентификации).

**Ответ:**
```json
{
  "users": {
    "total": 10,
    "online": 3
  },
  "tasks": {
    "total": 50,
    "pending": 15,
    "in_progress": 20,
    "completed": 10,
    "cancelled": 5
  },
  "messages": {
    "total": 500,
    "today": 25
  },
  "files": {
    "total": 100,
    "size_bytes": 1048576
  },
  "uptime": 3600
}
```

### Авторизованные endpoints

#### GET `/api/tasks`

Список задач (фильтрация по роли автоматически).

**Логика:**
- Врач видит только свои задачи
- Физик видит все задачи
- Админ видит все задачи

#### GET `/api/task/<id>`

Детали задачи.

**Доступ:**
- Врач-создатель: доступ есть
- Другие врачи: доступ запрещён (403)
- Физики: доступ есть
- Админ: доступ есть

#### POST `/create_task`

Создание задачи.

**Доступ:** Только врач

**Поля формы:**
- `title` — заголовок (опционально, авто-генерация)
- `description` — описание
- `study_type` — тип исследования (import_ct, import_ct_diag, mr, pet)
- `patient_card` — карта пациента (обязательно)
- `aria_availability` — доступность Aria
- `ct_building` — корпус для CT
- `ct_equipment` — аппарат CT
- `treatment_building` — корпус для лечения
- `treatment_device` — аппарат для лечения
- `breath_control` — контроль дыхания
- `expert_group` — экспертная группа
- `files` — файлы

#### POST `/update_task/<id>`

Обновление статуса задачи.

**Доступ:** Только физик или админ

**Поля:**
- `status` — новый статус

## Онлайн-статус

Пользователь считается онлайн, если `last_seen < 5 минут назад`.

Обновление `last_seen` происходит:
- При каждом запросе с аутентификацией (before_request)
- При входе в систему
