Безопасность и соответствие

Как Task-Chat соответствует требованиям к медицинским данным

Task-Chat (локально)

  • Сервер внутри сети клиники
  • Доступ только из локальной сети
  • Данные не передаются в интернет
  • SQLite база данных локально
  • Логи всех действий пользователей
  • Резервное копирование вручную

Облачные сервисы

  • Данные в облаке провайдера
  • Доступ из любой точки мира
  • Риск утечки данных
  • Зависимость от провайдера
  • Могут не соответствовать 152-ФЗ
  • Сложный аудит доступа

Важно для руководства

  • Task-Chat размещается только локально — на сервере клиники
  • Доступ к системе только из внутренней сети — нет доступа из интернета
  • Данные не передаются третьим лицам — полностью локальное хранение
  • Все действия пользователей логируются — возможен аудит
  • Резервное копирование контролируется администрацией клиники

Что НЕ делает Task-Chat

Честно о возможностях системы

Не импортирует пациентов

Task-Chat только передаёт информацию о пациентах физику. Сам импорт происходит во внешней системе планирования.

Не работает из интернета

Доступ только из локальной сети клиники. Это требование безопасности для медицинских данных.

Не заменяет систему планирования

Task-Chat — система коммуникации. Планирование лечения происходит во внешней системе (Aria и др.).

Не хранит DICOM-файлы

DICOM хранятся в системе отделения. В Task-Chat можно прикрепить только скриншоты и документы.

Что Task-Chat ДЕЛАЕТ

  • Фиксирует всех пациентов после исследования/разметки
  • Передаёт информацию физику для импорта
  • Связывает врачей и физиков через чат
  • Контролирует сроки обработки задач
  • Сохраняет историю коммуникации
  • Предоставляет статистику для отчётности
