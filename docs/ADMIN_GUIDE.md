## Первый вход

### Учётные данные по умолчанию

После установки системы:
- **Логин:** `admin`
- **Пароль:** `admin123`

### Обязательные действия после первого входа

1. **Смените пароль администратора**
   - Обратитесь к другому администратору (если есть) или используйте консоль
   - В production-среде сразу настройте безопасный пароль

2. **Создайте учётные записи для пользователей**
   - Перейдите в раздел **Админ → Пользователи**
   - Создайте учётные записи для врачей и физиков

## Управление пользователями

### Регистрация пользователя

1. Перейдите в **Админ → Пользователи**
2. Нажмите кнопку **«Добавить пользователя»**
3. Откроется модальное окно регистрации

**Поля формы:**

| Поле | Описание | Требования |
|------|----------|------------|
| **Имя пользователя** | Уникальный логин | 2-80 символов, уникальное |
| **Пароль** | Временный пароль | Минимум 3 символа |
| **Роль** | Врач или Физик | Обязательно |
| **Фамилия** | Фамилия пользователя | Обязательно |
| **Имя** | Имя пользователя | Обязательно |
| **Отчество** | Отчество | Опционально |
| **Отделение** | Выбор отделения | Зависит от роли |

**Отделения:**
- Для врачей: `РО1`, `РО2`, `РО3`, `РО4`, `РО5`
- Для физиков: `ФРО`

4. Нажмите **«Зарегистрировать»**

Пользователь появится в списке. Сообщите ему логин и временный пароль.

### Просмотр списка пользователей

1. Перейдите в **Админ → Пользователи**
2. Отобразится таблица всех пользователей

**Колонки таблицы:**
- **ID** — уникальный идентификатор
- **Имя пользователя** — логин
- **Роль** — значок роли
- **ФИО** — Фамилия И.О. (отделение)
- **Дата регистрации** — дата создания учётной записи
- **Был в сети** — последнее появление
- **Статус** — Онлайн/Офлайн
- **Действия** — кнопки управления

### Фильтрация пользователей

Используйте поиск для нахождения пользователя:
- По имени пользователя
- По ФИО
- По отделению
- По роли

### Смена пароля пользователя

1. Найдите пользователя в списке
2. Нажмите на иконку ключа 🔑
3. Откроется страница смены пароля
4. Введите новый пароль (минимум 3 символа)
5. Подтвердите пароль
6. Нажмите **«Изменить пароль»**

Сообщите новый пароль пользователю через защищённый канал связи.

### Удаление пользователя

⚠️ **Внимание:** Удаление пользователя приведёт к следующим последствиям:
- Задачи, созданные пользователем (если это врач), останутся в системе
- Сообщения пользователя в чатах сохранятся
- Файлы, загруженные пользователем, останутся в системе

**Рекомендация:** Перед удалением убедитесь, что у пользователя нет активных задач.

1. Найдите пользователя в списке
2. Нажмите на иконку корзины 🗑️
3. Подтвердите удаление в диалоговом окне

## Управление задачами

### Просмотр всех задач

1. Перейдите в **Админ → Задачи**
2. Отобразится таблица всех задач системы

**Колонки таблицы:**
- **ID** — номер задачи
- **Заголовок** — название задачи
- **Тип** — тип исследования (CT, MR, PET)
- **Карта пациента** — номер карты
- **Статус** — текущий статус
- **Врач** — создатель задачи
- **Дата создания** — когда создана
- **Действия** — кнопки управления

### Фильтрация задач

Используйте фильтры для поиска задач:
- По статусу (Ожидает, В работе, Выполнена, Отменена)
- По типу исследования
- По врачу
- По периоду создания

### Редактирование задачи

1. Найдите задачу в списке
2. Нажмите на иконку карандаша ✏️
3. Откроется страница редактирования

**Редактируемые поля:**

| Поле | Описание |
|------|----------|
| **Заголовок** | Название задачи |
| **Описание** | Детали задачи |
| **Статус** | pending, in_progress, completed, cancelled |
| **Приоритет** | low, medium, high, urgent |
| **Врач** | Исполнитель задачи |
| **Физик** | Ответственный физик |
| **Тип исследования** | import_ct, mr, pet |
| **Карта пациента** | Номер карты |
| **Aria доступность** | Новый / Уже есть |
| **Экспертная группа** | Для CT-исследований |
| **CT-диагностика** | Аппарат и корпус для CT |
| **Лечение** | Аппарат и корпус для лечения |
| **Контроль дыхания** | Тип контроля |
| **MR-диагностика** | Для MR-исследований |
| **PET-диагностика** | Для PET-исследований |

4. Нажмите **«Сохранить»**

### Удаление задачи

⚠️ **Внимание:** Удаление задачи приведёт к потере:
- Всех сообщений в чате задачи
- Всех прикреплённых файлов
- Истории изменений

**Рекомендация:** Вместо удаления используйте статус «Отменена».

1. Найдите задачу в списке
2. Нажмите на иконку корзины 🗑️
3. Подтвердите удаление в диалоговом окне

## Массовые операции

### Массовое изменение статуса

1. Перейдите в **Админ → Задачи**
2. Отметьте задачи галочками
3. В блоке **«Массовые действия»**:
   - Выберите действие: **«Изменить статус»**
   - Выберите новый статус
4. Нажмите **«Выполнить»**

Все отмеченные задачи получат новый статус.

### Массовое изменение врача

1. Отметьте задачи галочками
2. В блоке **«Массовые действия»**:
   - Выберите действие: **«Изменить врача»**
   - Выберите нового врача из списка
3. Нажмите **«Выполнить»**

Все отмеченные задачи будут переназначены выбранному врачу.

### Массовое удаление задач

⚠️ **Внимание:** Это действие необратимо!

1. Отметьте задачи галочками
2. В блоке **«Массовые действия»**:
   - Выберите действие: **«Удалить задачи»**
3. Нажмите **«Выполнить»**
4. Подтвердите удаление в диалоговом окне

Все отмеченные задачи будут удалены без возможности восстановления.

### Главная страница админ-панели

На странице **Админ** отображается статистика системы:

## Логи

### Расположение логов

```
data/logs/
├── taskchat.log           # Текущий лог
└── taskchat_history.log   # Архивные логи
```

### Категории логов

| Категория | События |
|-----------|---------|
| `auth` | Вход, выход, регистрация, смена пароля |
| `tasks` | Создание, обновление, удаление задач |
| `chat` | Отправка сообщений |
| `files` | Загрузка файлов |
| `admin` | Действия администратора |
| `system` | Системные события |

### Просмотр логов

#### Через файловую систему (Windows)

```powershell
# Просмотр текущего лога
Get-Content data\logs\taskchat.log -Tail 50 -Wait

# Поиск по логам
Select-String -Path data\logs\taskchat.log -Pattern "admin"
```

#### Через файловую систему (Linux)

```bash
# Просмотр в реальном времени
tail -f data/logs/taskchat.log

# Поиск по логам
grep "admin" data/logs/taskchat.log
```

### Формат логов

**JSON-формат (по умолчанию):**
```json
{
  "timestamp": "2026-03-12T10:30:00Z",
  "level": "INFO",
  "category": "auth",
  "message": "User logged in",
  "user_id": 1,
  "username": "admin"
}
```

**Текстовый формат:**
```
2026-03-12 10:30:00 | INFO      | auth       | User logged in
```

### Анализ логов

**Проверка неудачных попыток входа:**
```bash
grep "Failed login" data/logs/taskchat.log
```

**Проверка действий администратора:**
```bash
grep '"category": "admin"' data/logs/taskchat.log
```

**Проверка создания задач:**
```bash
grep "Task.*created" data/logs/taskchat.log
```

## Безопасность

### Требования к паролям

- Минимум 3 символа (системное требование)
- Рекомендуется: минимум 8 символов
- Рекомендуется: заглавные и строчные буквы, цифры, символы

### Смена пароля администратора

Если в системе один администратор:

1. Откройте консоль Python
2. Выполните команды:
```python
from app.app import create_app
from app.models import db, User

app = create_app()
with app.app_context():
    admin = User.query.filter_by(role='admin').first()
    admin.set_password('новый_пароль')
    db.session.commit()
```

### Мониторинг подозрительной активности

Проверяйте логи по категории `auth`:
- Многократные неудачные попытки входа
- Вход в нерабочее время

### Резервное копирование

**Что резервировать:**

| Компонент | Путь | Частота |
|-----------|------|---------|
| **База данных** | `data/instance/taskchat.db` | Ежедневно |
| **Файлы** | `data/uploads/` | Ежедневно |
| **Логи** | `data/logs/` | Еженедельно |

**Скрипт резервного копирования (Windows):**

```batch
@echo off
set BACKUP_DIR=E:\Backups\TaskChat
set DATA_DIR=E:\Projects\Task-chat-0-4-9\task-chat\data
set DATE=%DATE:~-4,4%%DATE:~-7,2%%DATE:~-10,2%

mkdir %BACKUP_DIR%
xcopy %DATA_DIR%\instance %BACKUP_DIR%\instance_%DATE% /E /I
xcopy %DATA_DIR%\uploads %BACKUP_DIR%\uploads_%DATE% /E /I
```

## Диагностика проблем

### Сервер не запускается

**Ошибка: Port 5000 already in use**

```bash
# Найти процесс
netstat -ano | findstr :5000

# Завершить процесс
taskkill /PID <PID> /F

# Или изменить порт
set PORT=5001
python app/run.py
```

### Ошибка базы данных

**Ошибка: Database is locked**

1. Остановите сервер
2. Удалите файл блокировки (если существует):
   ```
   data\instance\taskchat.db.lock
   ```
3. Запустите сервер

### Пользователь не может войти

**Ошибка: Invalid credentials**

1. Проверьте правильность ввода логина/пароля
2. Сбросьте пароль пользователя в админ-панели
3. Проверьте, не заблокирована ли учётная запись

### Файлы не загружаются

**Ошибка: File too large**

1. Проверьте размер файла (макс. 50 MB)
2. Проверьте свободное место на диске

**Ошибка: Unsupported file type**

Проверьте список разрешённых расширений в `config.py`.
